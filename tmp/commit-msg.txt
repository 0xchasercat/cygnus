fix: match daemon's serde snake_case kind "git_hub_status" in sanitizeGithubData

The Rust daemon serializes `AdminData::GitHubStatus` using serde's `rename_all = "snake_case"`,
which turns `GitHubStatus` into `"git_hub_status"` (with an underscore between `git` and `hub`).

`console/server.js`'s `sanitizeGithubData` function previously only checked `kind === "github_status"`.
Because `data.kind` was `"git_hub_status"`, the check failed, and `sanitizeGithubData` stripped the
`configured` and `app` fields from the API response payload, returning `{ kind: "git_hub_status" }`.

Consequently, `live.svelte.js` evaluated `d?.configured === true` as `false` (`undefined === true`),
leaving `this.github.configured = false` in the UI even after the GitHub App was stored in SQLite.

Updated `sanitizeGithubData` to accept `"git_hub_status"` as well as `"github_status"`, normalizing
the response to `"github_status"` with `configured` and `app` preserved. Added unit test.
