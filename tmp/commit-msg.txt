fix: allow extra query params on manifest callback URL

GitHub can append extra query parameters (such as `installation_id`,
`setup_action`, or tracking params) to the `redirect_url` when bouncing
the browser back after app registration. The `manifestCallback` handler
was strictly enforcing that ONLY `code` and `state` were in the query string
and returned HTTP 400 "unsupported fields" if any other parameter was present.
Removing the rigid whitelist allows `manifestCallback` to extract `code` and
`state` and exchange the credentials successfully regardless of extra query params.
