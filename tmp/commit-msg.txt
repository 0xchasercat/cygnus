fix: stop hiding the GitHub manifest form with display:none

The console's "Connect GitHub" flow builds a synthetic form, appends
it to document.body, and calls form.submit() to POST the manifest
JSON to github.com. The form was being hidden with form.hidden = true
so it would apply the HTML `hidden` attribute, which the browser styles
with display: none. Multiple browsers (Chrome, Safari, Firefox) will
not auto-submit a form whose computed display is none, so the
.github.com POST never happened. The backend manifestStart succeeded
(server returned 200 with action+manifest), but the browser never
left the console, so the operator saw a silent "nothing happened".

The HAR trace shows the manifest POST returning 200 with a 644-byte
body, then 40+ seconds of nothing before the next poll — no POST to
github.com, no callback to /github/app/manifest/callback. That 40
seconds of silence is the user staring at a form that was never going
to submit.

Fix: position the form off-screen with inline styles instead of the
hidden attribute, and wrap form.submit() in a try/catch so a blocked
submission surfaces a clear error instead of a silent failure.
