fix: force immediate GitHub status fetch on callback redirect

When returning from GitHub App setup or installation, `#handleGithubCallback`
invoked `#poll()`, but `#poll()` only fetched `/api/v1/github/status` on every
3rd tick (`this.#tick % GITHUB_EVERY === 0`). Because `#tick` was usually not
divisible by 3 on the single forced poll, the status fetch was skipped, leaving
`this.github.configured = false` and causing the UI to render the unconfigured
"Connect GitHub" form.

Added `refreshGithub()` which immediately fetches status, repositories, and jobs
in parallel, and call it directly from `#handleGithubCallback()`. Also supported
`#poll(forceGithub = true)` to bypass the cadence check when needed.
