ux: add direct "Install App on GitHub" button to eliminate manual Installation ID entry

Creating a GitHub App from a manifest registers the App in GitHub and Cygnus,
but does not automatically install it on a user/org account until the operator
completes the GitHub installation step.

Previously, the UI fell back to a manual numeric input for "Installation ID".
Added a direct `[Install App on GitHub ->]` primary action linking to
`store.github.app.html_url`. Clicking the button opens GitHub's installation
picker directly; upon clicking "Install", GitHub redirects back to Cygnus at
`/?github=setup&installation_id=...`, which automatically auto-discovers and
populates the repositories list without requiring any manual copy-pasting.
